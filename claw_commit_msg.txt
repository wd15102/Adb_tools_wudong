fix: 移除所有 ADB shell 命令中的管道和重定向，修复 Windows 兼容性

背景
---
collector.py 中的 shell 命令使用了 2>/dev/null、| head -200 等操作符。
在 Windows 上 subprocess.Popen(..., shell=True) 会由本地 cmd.exe 预处理：
- 2>/dev/null → Windows 找不到 /dev/null 路径
- | head -200 → Windows 无 head 命令
- | grep/findstr → cmd.exe 本地拦截管道而非发送到设备

改动
---
- _get_all_pids():「adb shell ps | grep pkg」→ adb shell ps + Python 解析过滤
- _collect_fd(): 去掉 2>/dev/null
- _collect_threads(): 去掉 2>/dev/null
- _collect_fps(): 去掉 2>/dev/null | head -200, Python 切片 [:200] 替代
- 全部改为纯 adb shell 命令 + Python 端过滤，零 shell 操作符依赖
