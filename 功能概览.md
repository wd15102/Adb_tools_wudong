# 🥭 AdbTool 功能概览

> 芒果TV · Android 机顶盒/大屏应用测试工具  
> 作者：吴东 · wudong@mgtv.com

---

## 📋 工具简介

AdbTool 是一款专为 **芒果TV 机顶盒/大屏应用** 设计的综合测试工具，集成了性能监控、自动化测试、抓包分析、数据上报验证等多种功能。支持 Windows/macOS 双平台，提供 GUI 界面和 Web 仪表盘两种操作方式。

---

## 🎯 核心功能模块

### 1. 性能测试（Performance Testing）

#### 1.1 传统性能测试
- **实时监控指标**：CPU、内存、线程数、FD（文件描述符）、FPS
- **测试模式**：指定设备测试 / 全部设备批量测试
- **数据存储**：SQLite 数据库 + Excel 报告
- **崩溃监控**：自动检测 ANR、Crash、PlayerError
- **日志采集**：自动拉取 logcat、ANR traces、dmesg

#### 1.2 Web 实时仪表盘（Web Dashboard）
- **技术栈**：Flask + SocketIO + ECharts
- **默认端口**：5050
- **功能特性**：
  - 实时曲线展示（CPU、内存、FPS）
  - 自动检测 ADB 设备和包名
  - 历史数据对比
  - 崩溃自动告警
  - 数据持久化到 `results/perf_dashboard.db`

---

### 2. 抓包与 Mock（Capture & Mock）

#### 2.1 抓包模块
- **技术栈**：mitmproxy 12.2.2 + Flask + SocketIO
- **代理端口**：8888（HTTP/HTTPS）
- **Web UI 端口**：8899
- **功能特性**：
  - Charles 风格界面（分组树、请求/响应详情）
  - 内存环形缓冲区（500 条，不存数据库）
  - 全局搜索（URL、请求体、响应体）
  - 中文 Unicode 自动解码（`\uXXXX` → 中文）
  - 设备流量过滤（按客户端 IP）
  - 请求高亮和跳转

#### 2.2 Mock 功能
- **5 种 Mock 模式**：
  1. 修改响应状态码
  2. 修改响应 Headers
  3. 修改响应 Body
  4. 请求转发（Map Remote）
  5. 请求/响应延迟
- **规则持久化**：SQLite 存储 Mock 规则
- **实时生效**：无需重启代理

#### 2.3 API 文档集成
- **Confluence 文档抓取**：自动抓取 IPTV CMS-EPG 接口规范
- **智能匹配**：根据 host + path + method 自动匹配 API 文档
- **参数解析**：请求参数自动填充到 API 文档模板

---

### 3. 数据上报验证（Data Report Validation）

#### 3.1 上报拦截与验证
- **拦截域名**：`henanyddpproxy.cslgtest.imgo.tv` 等
- **验证内容**：
  - 字段完整性检查
  - 字段值比对（与预期值对比）
  - 时间戳合法性
  - 字段长度校验
  - 关联字段一致性
- **支持项目**：OTT、IPTV、移动、电信、联通

#### 3.2 上报字段覆盖
- **公共字段**：did、aver、bid、time、uuid 等
- **业务字段**：vid、cid、cntp、lastp、lob 等
- **自定义字段**：支持配置文件扩展

---

### 4. 设备操作（Device Operations）

#### 4.1 设备信息查询
- 系统版本（`ro.build.version.release`）
- SDK 版本（`ro.build.version.sdk`）
- 设备 MAC 地址（多网卡自动回退）
- 设备厂家/型号/序列号

#### 4.2 设备控制
- 输入文字（支持中文）
- 执行 Shell 命令
- 文件拉取/推送
- Ping 网络测试
- Host 配置管理
- 查看 CPU TOP 进程

#### 4.3 遥控器模拟
- 方向键（上/下/左/右）
- 确认键、返回键、Home 键
- 菜单键、音量键
- 数字键（0-9）

---

### 5. APK 操作（APK Operations）

#### 5.1 包信息查询
- 包信息（`dumpsys package`）
- 包路径（`pm path`）
- 版本号/版本代码
- 内存占用（`dumpsys meminfo`）
- 进程信息（PID、线程数）
- 文件描述符（FD）统计

#### 5.2 内存分析
- 实时内存监控
- 内存 Dump（HPROF）
- 内存泄漏初步检测

#### 5.3 Activity 管理
- 当前 Activity 查询
- Activity 历史记录
- 黑/白名单 Activity 过滤

---

### 6. 批量操作（Batch Operations）

- **批量安装**：多设备安装 APK
- **批量卸载**：多设备卸载应用
- **批量重启**：多设备重启
- **批量启动**：多设备启动应用
- **批量按键**：多设备同步遥控器操作

---

### 7. 版本切换（Version Switch）

支持多运营商版本快速切换：
- **电信**：发布版 / 预发布版
- **移动 IPTV**：发布版 / 预发布版
- **移动 OTT**：发布版 / 预发布版
- **联通**：发布版 / 预发布版

---

### 8. UI 测试（UI Testing）

#### 8.1 兼容性测试
- 截图对比（Scrcpy / Minicap）
- XML 布局对比
- 分辨率适配检测

#### 8.2 投屏工具
- **Scrcpy**：高质量投屏 + 控制
- **Minicap**：轻量级截图

---

### 9. Monkey 测试（Monkey Testing）

- **自定义策略**：
  - `--pct-majornav 40 --pct-nav 50 --pct-syskeys 10`
  - `--pct-touch 60 --pct-motion 40`
- **事件间隔**：可配置 throttle
- **崩溃捕获**：自动保存崩溃日志
- **ANR 检测**：自动拉取 ANR traces

---

### 10. 日志工具（Log Tools）

#### 10.1 Logcat 采集
- 实时日志显示
- 关键字过滤
- 日志级别筛选
- 崩溃自动高亮

#### 10.2 日志看板（Log Board）
- 高性能日志显示（虚拟列表）
- 支持大日志文件（>100MB）
- 实时搜索和过滤

#### 10.3 崩溃日志管理
- 自动分类存储（`crash_logs/YYYYMMDD/`）
- 关联 logcat / ANR / dmesg
- 崩溃趋势统计

---

### 11. 网络工具（Network Tools）

#### 11.1 HTTP 代理
- **本地代理**：修改响应数据
- **远程代理**：请求转发到测试环境
- **配置热更新**：支持 `proxy.json` 动态加载

#### 11.2 Curl 请求
- 可视化 Curl 请求构建
- 请求历史记录
- 响应格式化显示

#### 11.3 短信清理
- 自动清理测试短信
- 短信内容验证

---

### 12. 报告与数据（Reporting）

#### 12.1 性能报告
- Excel 格式报告
- 图表展示（CPU、内存趋势）
- 崩溃统计
- 测试摘要

#### 12.2 数据导出
- 数据库导出（SQLite → Excel）
- 日志打包下载
- 截图批量导出

#### 12.3 邮件通知
- 测试完成自动邮件
- 崩溃告警邮件
- 报告附件发送

## 🏗️ 技术架构

```
AdbTool/
├── adbtool.py              # 主入口
├── config.py               # 全局配置
├── src/
│   ├── gui.py              # Tkinter GUI
│   ├── adbtools.py         # ADB 工具封装
│   ├── perf_test.py        # 性能测试核心
│   ├── proxy.py            # mitmproxy 代理
│   ├── addon.py            # 代理插件（数据上报）
│   ├── data_auto.py        # 数据上报验证
│   ├── logcat.py           # Logcat 采集
│   ├── web_dashboard/      # Web 仪表盘
│   │   ├── server.py       # Flask 服务器
│   │   ├── collector.py    # 数据采集器
│   │   └── templates/      # 前端页面
│   ├── capture/            # 抓包模块
│   │   ├── capture_server.py   # Flask 服务器
│   │   ├── capture_addon.py    # mitmproxy 插件
│   │   └── templates/      # 前端页面
│   └── ...
├── results/                # 测试结果
├── crash_logs/             # 崩溃日志
├── logs/                   # 运行日志
└── tools/                  # 工具资源
```
