# 抓包 & Mock 工具 (类似 Charles Proxy)

## 快速使用

### 1. 启动

**方式一（推荐）：从 AdbTool GUI 菜单启动**
```
性能测试 → 📡 抓包 & Mock
```

**方式二：命令行启动**
```bash
python src/capture/capture_server.py --port 5051 --proxy-port 8888
```

### 2. 设置设备代理

启动后会在浏览器打开 Web UI，顶部显示：
```
监听: 192.168.x.x:8888
```

在机顶盒/安卓设备的 **WiFi 设置 → 代理 → 手动**：
- **主机名**: 192.168.x.x（你电脑的 IP）
- **端口**: 8888

### 3. HTTPS 抓包（可选）

需要安装 mitmproxy 的 CA 证书到设备：
1. 设备浏览器访问 `http://mitm.it`（需在代理状态下）
2. 下载 Android 证书并安装
3. 或手动安装: `adb push ~/.mitmproxy/mitmproxy-ca-cert.pem /sdcard/`

### 4. Web UI 功能

| 区域 | 功能 |
|------|------|
| **左侧列表** | 实时显示所有 HTTP/HTTPS 请求（方法、域名、路径、状态码、耗时） |
| **右侧详情** | 点击请求可查看/复制：URL、请求头、请求体、响应头、响应体 |
| **🎯 Mock 规则** | 边栏管理 Mock 规则，支持 5 种动作 |
| **搜索** | 按域名或路径过滤请求 |

### 5. Mock 规则类型

| 动作 | 说明 | 示例 |
|------|------|------|
| **Map Local** | 拦截接口，返回自定义内容 | 把广告接口返回空数据 |
| **Map Remote** | 请求重定向到其他 URL | 切换到测试环境 |
| **修改状态码** | 修改 HTTP 响应码 | 返回 404/500 测试容错 |
| **注入延迟** | 人为增加响应时间 | 模拟弱网 3 秒延迟 |
| **丢弃请求** | 直接丢弃请求（不返回任何内容） | 屏蔽特定请求 |

## 端口说明

- **5051**: Web UI（浏览器访问）
- **8888**: 代理端口（设备设置用）
