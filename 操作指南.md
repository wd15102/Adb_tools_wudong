# 🥭 AdbTool 操作指南

> 芒果TV · Android 机顶盒/大屏应用测试工具  
> 快速上手与常用操作说明

---

## 📥 安装与启动

### 1. 环境准备

```bash
# 1. 安装 Python 3.8+
# 2. 安装 ADB（Android SDK Platform Tools）
# 3. 安装依赖
pip install -r requirements.txt
```

### 2. 启动工具

```bash
# 方式1：直接运行
python adbtool.py

# 方式2：使用打包版本（Windows）
./adbtool.exe
```

## 🔌 设备连接

### 连接机顶盒

1. **有线连接**：USB 线连接电脑和机顶盒
2. **无线连接**：
   ```bash
   # 先有线连接，然后执行
   adb tcpip 5555
   adb connect <设备IP>:5555
   ```

### 验证连接

在 AdbTool 主界面，查看设备列表是否显示设备。如果没有显示，点击菜单栏 **ADB → 刷新设备**。

## 📊 性能测试

### 启动 Web 仪表盘（推荐）

1. 点击菜单栏 **性能测试 → 🌐 Web仪表盘**
2. 浏览器自动打开 `http://127.0.0.1:5050`
3. 选择设备和包名，点击「开始采集」

#### 仪表盘功能

| 功能 | 操作 |
|------|------|
| 开始采集 | 选择设备 + 包名 → 点击「开始采集」 |
| 停止采集 | 点击「停止采集」 |
| 查看历史 | 点击「历史任务」标签 |
| 数据对比 | 选择多个任务 → 点击「对比模式」 |
| 崩溃告警 | 自动检测，红色高亮显示 |

### 传统性能测试

1. 点击菜单栏 **性能测试 → 指定设备/全部设备**
2. 选择监控指标（CPU、内存、FPS 等）
3. 点击「开始测试」

点击菜单栏 **性能测试 → 停止测试**，或按 `Ctrl+C`。

## 📡 抓包与 Mock

### 启动抓包模块

1. 点击菜单栏 **性能测试 → 📡 抓包 & Mock**
2. 浏览器自动打开 `http://127.0.0.1:8899`
3. 确保机顶盒已配置代理：
   - 代理地址：`172.31.111.166:8888`（或你的电脑 IP）
   - 代理类型：HTTP

### 机顶盒配置代理

```bash
# 在机顶盒上执行（需要 root）
settings put global http_proxy 172.31.111.166:8888

# 清除代理
settings put global http_proxy :0
```

### 抓包界面操作

| 功能 | 操作 |
|------|------|
| 查看请求 | 点击左侧列表中的请求 |
| 查看详情 | 右侧显示 Headers、Body、API 文档 |
| 搜索 | 顶部搜索框输入关键词，选择搜索范围 |
| 清空列表 | 点击「清空」按钮 |
| 暂停捕获 | 点击「暂停」按钮 |

### 创建 Mock 规则

1. 右键点击某个请求 → **创建 Mock 规则**
2. 选择 Mock 类型：
   - **修改状态码**：如将 200 改为 500
   - **修改响应头**：添加/修改 Headers
   - **修改响应体**：替换返回的 JSON 数据
   - **请求转发**：将请求转发到其他服务器
   - **添加延迟**：模拟网络延迟
3. 填写规则内容，点击「保存」
4. 规则立即生效，匹配的请求会显示 Mock 标签

### 管理 Mock 规则

- 点击「Mock 规则」标签查看所有规则
- 支持启用/禁用、编辑、删除规则

## 📤 数据上报验证

### 开启数据上报

1. 在 AdbTool 主界面，点击「数据上报」开关
2. 确保机顶盒网络正常
3. 打开 Charles 或其他抓包工具验证

### 查看上报数据

- 数据上报流量会经过 AdbTool 代理
- 在 Charles 中查看 `henanyddpproxy.cslgtest.imgo.tv` 域名
- 上报字段会自动填充 did、aver、bid 等公共字段

### 上报字段说明

| 字段 | 说明 |
|------|------|
| did | 设备唯一标识（MAC 地址或序列号） |
| aver | App 版本号 |
| bid | 业务线 ID |
| time | 时间戳（毫秒） |
| uuid | 请求唯一标识 |
| vid | 视频 ID |
| cid | 频道 ID |

## 🐛 常见问题

### 1. 设备连接失败

- 检查 USB 线是否正常
- 执行 `adb devices` 查看设备是否识别
- 重启 ADB 服务：`adb kill-server && adb start-server`

### 2. 抓包没有数据

- 检查机顶盒代理配置是否正确
- 检查防火墙是否放行 8888 端口
- 确认 mitmproxy 证书已安装到机顶盒

### 3. Web 仪表盘无法访问

- 检查 5050 端口是否被占用
- 查看控制台是否有错误日志
- 尝试重启 AdbTool

### 4. 数据上报 did 字段为空

- 检查设备 MAC 地址是否能正常读取
- 工具会自动回退到序列号作为 did

### 5. 中文显示乱码

- 确保所有配置文件使用 UTF-8 编码
- Windows 控制台默认 GBK，建议使用 Web 仪表盘
