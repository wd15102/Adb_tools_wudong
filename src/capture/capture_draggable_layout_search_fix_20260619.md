# 抓包面板布局重构 & 中文搜索修复

**时间：** 2026-06-19 ~12:48–13:xx

## 改动清单

### 1. 响应体默认完整显示（不再截断）
- **`capture_server.py`** — 截断阈值从 3000 提高到 102400 (100KB)
- 一般 JSON 响应不会再出现截断，前端不再需要「展开全部」按钮
- 超 100KB 的超大响应仍保留截断提示

### 2. 主面板布局 → 可拖拽三面板（仿 Charles）
- **左侧树（flow-tree）** ← 可拖拽分隔条（竖） → **右侧上（detail-panel）**
- **右侧上（detail-panel）** ← 可拖拽分隔条（横） → **右侧下（resp-bottom-panel）**
- 分隔条 hover 变蓝（#58a6ff），drag 时加 active 类保持蓝色
- 左侧宽度和底部高度保存在 `localStorage`，刷新后恢复

### 3. 响应体移至独立底部面板（可拖拽调整高度）
- 选中请求后，响应体不再作为 detail-panel 内可折叠 section 展示，而是独立在底部面板
- 底部面板有一个 **搜索栏 + 上一页/下一页按钮**，实时 DOM 搜索
- 底部面板默认隐藏（`display:none`），有响应体时显示为 `display:flex`
- 底部面板顶部有 **Body / Raw** Tab 切换（初版仅实现 Body）

### 4. 中文搜索修复（核心改动）
**根因：** 旧的 `searchInJsonHtml()` 用正则表达式在已语法高亮的 HTML 上搜索，`[^<>]*` 遇到 `<span>` 标签边界就断了，中文全被标签包围，永远搜不到。

**方案：** 新的 `_highlightInDom(rootEl, keyword)` 函数采用 **DOM TreeWalker** 方案：
- 创建 `NodeFilter.SHOW_TEXT` 遍历器，获取所有纯文本节点
- 对每个文本节点，用 `indexOf()` 不区分大小写匹配关键词（中文/英文/数字均支持）
- 使用 `document.createDocumentFragment()` + `<mark>` 元素替换匹配区域
- 搜索前先清除旧高亮，保证增量搜索正确
- 刷新时恢复原始 HTML 再重新高亮

### 5. UI 精简
- 删除废弃的 `expandFullResp()` 函数（不再需要展开全部按钮）
- 移除内联的响应体 section 和搜索栏（已迁移到底部独立面板）
